## 重点テスト項目（ビルド14の修正内容）

### 1. シングルモード グループ作成後のデフォルトリスト自動作成

修正内容: SingleGroupCreationDialog のmountedチェック削除、および SharedListPage でグループにリストが0件のとき「買い物リスト」を自動作成する

- [ ] 新規サインアップ後に初めてグループを作成できる
- [ ] グループ作成直後に「買い物リスト」が自動生成されている
- [ ] 「買い物リスト」が自動的にカレントリストとして選択されている
- [ ] アイテム追加ボタンをタップするとアイテムを追加できる（「リストを選択」エラーが出ない）
- [ ] GroupCreationWithCopyDialog 経由でグループを追加した場合も「買い物リスト」が自動生成される

### 2. シングルモード FAB グレーアウト

修正内容: シングルモードでグループが1件以上あるとき、グループ追加FABとQRスキャンFABをグレーアウトして操作不可にする

- [ ] グループが0件のとき FAB が青色で有効になっている
- [ ] グループを1件作成した後 FAB がグレーアウトになっている
- [ ] グレーアウト中に FAB をタップしても何も起きない（グループ作成ダイアログが開かない）
- [ ] グレーアウト中に QR スキャン FAB をタップしても何も起きない
- [ ] マルチモードではグループ数に関係なく FAB が常時有効になっている

### 3. 初回グループ作成の赤画面根本解消（InitialSetupWidget廃止）

修正内容: SharedGroupPage から InitialSetupWidget を完全削除し、常に GroupListWidget を表示する

- [ ] 新規サインアップ後の初回グループ作成で赤画面が出ない
- [ ] グループ作成後に GroupListWidget が正常に表示される
- [ ] グループ0件状態でアプリを開いたとき空状態の案内テキストが表示される

### 4. アカウント削除スピナー残留修正

修正内容: アカウント削除処理完了後にローディングダイアログが閉じない問題を修正

- [ ] アカウント削除を実行するとスピナーが表示される
- [ ] 削除処理完了後にスピナーが自動的に閉じる
- [ ] 削除完了後にサインイン画面へ遷移できる
- [ ] キャンセルした場合もスピナーが残留しない

### 5. アカウント削除 再認証ダイアログのレイアウト崩れ修正

修正内容: 小画面端末や文字スケール変更時に再認証ダイアログがオーバーフローしていた問題を修正

- [ ] アカウント削除の再認証ダイアログが正常に表示される
- [ ] ダイアログ内のボタンや文字が画面外にはみ出していない
- [ ] 小さい画面（SH-54D など）でもダイアログが読める

### 6. QR招待クロスデバイス動作確認

修正内容: iOS prod が古い Firebase プロジェクトを参照していたため Android prod でスキャンすると「招待が見つかりません」エラーになっていた問題を修正

- [ ] iOS 端末で QR コードを発行できる
- [ ] 発行した QR コードを Android 端末でスキャンできる
- [ ] スキャン後にグループへの参加が完了する（「招待が見つかりません」エラーが出ない）
- [ ] 参加後にグループがリストに表示される
- [ ] 逆方向（Android で発行、iOS でスキャン）も正常に動作する

### 7. watchUserGroups リアルタイム同期修正

修正内容: watchUserGroups() が旧コレクション名を参照していたためリアルタイム同期が機能しなかった問題を修正

- [ ] グループを作成すると他デバイスで3秒以内に表示される
- [ ] グループ名を変更すると他デバイスに反映される
- [ ] メンバーが参加するとグループ一覧に反映される
