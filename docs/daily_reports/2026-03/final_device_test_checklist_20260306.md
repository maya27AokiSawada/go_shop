**重要度の定義**:

- **P0 (Critical)**: アプリクラッシュ、データ損失、セキュリティ問題
- **P1 (High)**: 機能不全、同期失敗、認証エラー
- **P2 (Medium)**: UI不具合、パフォーマンス劣化、軽微な表示問題

## 総合判定

□ **合格**: 全P0/P1項目が合格かつ合格率90%以上
□ **条件付き合格**: P0項目は全合格、P1の一部に問題あり
□ **不合格**: P0に問題あり、または合格率70%未満

## テスト実施ガイド

### 推奨テスト順序

1. **セクション1 (認証)** → セクション2 (ホーム) → まず基本動作を確認
2. **セクション3 (グループ)** → セクション4 (買い物リスト) → コア機能を確認
3. **セクション5 (ホワイトボード)** → 描画機能を集中テスト
4. **セクション8 (ネットワーク)** → 詳細手順は `network_monitor_test_20260304.md` 参照
5. **セクション9 (マルチデバイス)** → 2台以上で実施
6. **セクション6-7, 10-12** → 残りのテストを実施

### テスト実施の注意点

1. **認証テスト**: サインアウト→サインインのサイクルで前ユーザーのデータが残らないこと確認
2. **マルチデバイス同期テスト**: 最低2台のデバイスで同時にアプリを開いた状態で実施
3. **ネットワークテスト**: 機内モードの切り替えだけでなく、WiFi OFF→ON のサイクルもテスト
4. **ホワイトボードテスト**: 描画と保存のタイミングを注意深く観察、Firestore + Hive両方同期を確認
5. **QRスキャンテスト**: 十分な照明下で実施（暗い環境ではスキャン精度が低下）
6. **エラーログ確認**: テスト終了後、設定→エラー履歴で予期しないエラーがないか確認
