### 2.4 fire-and-forget + Firestore オフライン永続化による保存

> **実装内容**: 保存処理は `unawaited`（fire-and-forget）で Firestore SDK に発火する。
> Firestore のオフライン永続化により、ネットワーク切断中でも**ローカルキャッシュへの書き込みは即時成功**し、ユーザーには「保存しました」が表示される。
> サーバーへの実際の同期は Firestore SDK がバックグラウンドで自動処理し、ネットワーク回復後に他端末へ反映される。
>
> また、`PersonalWhiteboardCacheService`（SharedPreferences）には**ストローク全体と未保存 strokeId を永続化**するため、
> 描画したまま保存せずに閉じた場合でも再入時に未保存状態が復元される。

- [x] Pixel 9（まや）端末で個人用ホワイトボードを開く
- [x] 描画を追加する
- [x] ネットワークを切断する
- [x] 保存ボタンを押す
- [x] **「保存しました」が即時表示される**（Firestore ローカルキャッシュへの書き込み成功）
- [x] 保存ボタンが即時にアクティブ状態に戻る
- [x] logcat に `✅ [SAVE] ローカル書き込み発火:` が出る
- [x] logcat に `❌ [SAVE] バックグラウンド保存エラー` が出ない
- [x] ネットワークを復旧する
- [x] 数秒以内に他端末でも描画が反映される（Firestore SDK が自動同期）

**サブシナリオ: オフライン中に画面を閉じた場合の再入復元**

> 閉じるときは `_navigateToGroupDetail` 内で自動保存（`_saveWhiteboard(silent: true)`）が実行されるが、
> オフライン状態のときはオフラインチェックでブロックされ保存がスキップされる。
> その後 `_syncPersonalWhiteboardCache` が呼ばれ、未保存 strokeId がキャッシュに保存される。

- [x] 描画を追加する

- [x] 再度閉じて開いたとき、同じ stroke が未保存のまま残留しない

結果: メインシナリオ・サブシナリオとも OK（仕様通り）

備考: サブシナリオでネット断後に閉じると auto-save が fire-and-forget で Firestore ローカルキャッシュに成功（メインシナリオと同じ flow）。PersonalWhiteboardCacheService の pending 復元パスは NetworkMonitor がオフライン検知済みの状態でのみ発動するフォールバック設計。今回の挙動は仕様通り。
