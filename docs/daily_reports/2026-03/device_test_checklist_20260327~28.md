- [x] 端末 A で保存 → 端末 B で反映される（数秒程度）
- [x] 複数回繰り返し保存してもグレーアウト固着しない
- [x] logcat に `✅ [SAVE] ローカル書き込み発火` が出て `❌` が出ない

### 2.2 ストロークサブコレクション監視

> **仕様変更**: `watchStrokesSubcollection` でリアルタイム監視。
> `orderBy` を廃止しクライアントソートに変更（Firestore インデックス不要）。
> `hasPendingWrites` フィルターを廃止（fire-and-forget 対応）。

- [x] ホワイトボードを開いたときに描画内容が正しく読み込まれる
- [x] 他端末が保存したストロークが数秒以内に反映される
- [x] logcat に `📡 [WATCH_STROKES] スナップショット受信` が出ている
- [x] logcat に `❌ [WATCH_STROKES]` エラーが出ていない
- [x] 開くたびに重複ストロークが増えない

結果: OK

備考: 再テストで問題なし。logcat で `📡 [WATCH_STROKES] スナップショット受信` を確認し、`❌ [WATCH_STROKES]` は確認されず。自端末再オープン時の stale cache 問題は個人用ホワイトボードキャッシュのクリーンアップ追加後に再発なし。

### 2.3 全消去の他端末反映修正

> **修正内容**: Firestore から空のストローク一覧を受けたとき、他端末のローカル保存済み stroke を復活させないようにマージ条件を修正。

- [x] 端末 A で既存描画があるボードを開く
- [x] 端末 B でも同じボードを開き、同じ描画が見えていることを確認
- [x] 端末 A で全消去を実行
- [x] 端末 A で即時に描画が消える
- [x] 端末 B でも数秒以内に描画が消える
- [x] 端末 B を閉じて再度開いても消去状態のまま
- [x] 全消去直後に旧描画が一瞬だけでも復活しない
- [x] logcat に `🗑️ [REPO] サブコレクション全消去開始` と `✅ [REPO] 全消去完了` が出る
- [x] logcat に `🔄 [MERGE]` が出ても、消去後の結果本数が 0 本のまま

結果: OK

備考: 端末 A=TBA1011、端末 B=Pixel 9 で再テスト。端末 A の logcat で `🗑️ [REPO] サブコレクション全消去開始` と `✅ [REPO] 全消去完了` を確認。端末 B の logcat で `📡 [WATCH_STROKES]` 受信後に `🔄 [MERGE]` が Firestore=0 本で反映され、消去後の再復活は確認されず。
