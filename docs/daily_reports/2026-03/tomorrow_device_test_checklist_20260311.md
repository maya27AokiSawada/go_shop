## 3. 最優先確認: ネットワーク回復同期

### 3.1 リスト更新の回復同期

- [x] 端末 A をオフラインにする
- [x] 端末 A で既存リストにアイテム追加を行う
- [x] 端末 A で既存リストのアイテム購入状態を変更する
- [x] オンライン復帰後、端末 B に変更が反映される
- [ ] Firestore 側にも最終状態が保存される

結果: 購入状態がオンライン復帰後に他端末へ反映されることは確認。

課題: 2 番目の手順でアイテム追加ダイアログを開いたままリトライしているようで、ダイアログが開きっぱなしになる。3 秒程度で UI 側は諦めてダイアログを閉じ、リトライ自体はリポジトリ層に任せて Hive のみで継続動作させる方がよい。

### 3.2 リストの新旧比較

- [x] 端末 A と B で同じリストを開く
- [x] 端末 A をオフラインにして変更する
- [x] その間に端末 B でも別変更を入れる
- [x] 端末 A をオンラインに戻す
- [x] 最終結果が古い状態で上書きされない
- [x] 明らかに新しい更新時刻の側が採用される

結果: OK。両端末で購入済みにしたアイテムを B で未購入に戻し、その後 A で未購入 → 購入に変更。A をオンライン復帰すると両端末とも購入済みになり、A 側の最新変更が採用されたように見える。

### 3.3 グループ更新の回復同期

- [x] 端末 A をオフラインにする
- [x] 端末 A でグループ名変更またはメンバー状態変化を発生させる
- [x] オンライン復帰後、端末 B に反映される
- [ ] Firestore にも同じ内容が反映される

結果: グループ名変更は他端末へ即時反映されることを確認済み。通知送信追加により、従来の「両端末オンラインでも即時反映しない」問題は解消。

補足: この項目は本来「オフライン復帰後の反映」と「Firestore への最終反映」まで確認して完了とする。今回確認できたのは即時反映までで、Firestore 側の最終状態確認は未実施。

UXメモ: オフライン中のグループ名変更では、グループ名テキストボックスは即時反映されたが、グループ詳細画面のアップバータイトルはオンライン復帰後の反映となった。低優先の表示遅延として扱う。

## 4. 最優先確認: 削除整合性

### 4.1 グループ削除の通常系

- [x] オーナー端末でグループ削除を実行する
- [x] オーナー端末で対象グループが消える
- [x] メンバー端末でも対象グループが消える
- [x] 通知履歴または画面反映で削除が確認できる

結果: OK 確認。

### 4.2 削除トゥームストーン優先

- [x] 端末 A で対象グループを表示したままオフラインにする
- [x] 端末 B でそのグループを削除する
- [x] 端末 A 側で古い画面状態のまま何らかの操作をしてからオンラインに戻す
- [x] 削除済みグループが復活しない
- [x] オンライン復帰後、端末 A でも削除済みとして扱われる

結果: OK。手順 3 でリスト作成を試したところ Hive には作成されているようだが、UI には表示されない。同名リスト作成時にバリデーションで弾かれるため Hive 側には残っていると判断できる。テスト観点としては、他端末で削除済みかつオフライン中に変更されたグループでは削除が優先される。

### 4.3 削除後の再作成・再参加

- [x] 削除したグループ名と同名の新規グループを作成できるか確認する
- [x] 旧グループの残骸が UI に混在しない
- [x] メンバー側で旧グループの選択状態が残らない

結果: テスト項目としては OK。

改善案: オフライン中に削除されたグループが UI 上では削除済みと分からないため、タイル色変更などでユーザーに状態を知らせた方がよい。
