# ユーザー切替確認チェックリスト - 2026-03-09

## 目的

サインアウト後に別ユーザーでサインインした際、前ユーザーのグループデータ・選択状態・最終使用リスト情報が残らないことを確認する。

---

## 対象修正

- ユーザー切替処理の単一路線化
- サインアウト時のローカル状態クリーンアップ統一
- `selected_group_id` / `current_list_id` / `group_list_map` など永続選択状態の削除
- Windows の user-specific Hive 切替の整合性修正

---

## 事前準備

- [✅] テスト用アカウント A を用意する
- [✅] テスト用アカウント B を用意する
- [✅] A と B で所属グループが明確に異なる状態を作る
- [✅] 可能なら A 専用グループ、B 専用グループ、共通グループを 1 つずつ用意する
- [✅] Windows 実行環境でアプリを起動できる状態にする
- [✅] Android 実機でも確認する場合は対象端末を接続する

---

## 期待する正しい動作

- [✅] 別ユーザーに切り替えた直後、前ユーザー専用グループが 1 件も表示されない
- [✅] 共通グループのみは両ユーザーで表示される
- [✅] 前ユーザーが最後に選択していたグループが新ユーザーに自動適用されない
- [ ] 前ユーザーが最後に開いていた買い物リストが新ユーザーに自動適用されない
- [ ] サインアウト後に再ログインしても、現在のユーザーに属するグループだけが表示される

---

## テストケース 1: A → ログアウト → B

### 手順

- [ ] A でサインインする
- [ ] グループ一覧を開く
- [ ] A 専用グループ名を確認する
- [ ] A 専用グループを選択する
- [ ] そのグループ内で任意の買い物リストを開く
- [ ] ログアウトする
- [ ] B でサインインする
- [ ] グループ一覧を開く

### 確認項目

- [ ] A 専用グループが表示されない
- [ ] B 専用グループが表示される
- [ ] 共通グループがある場合は表示される
- [ ] 初期選択グループが A の最終選択グループになっていない
- [ ] 初期表示リストが A の最終使用リストになっていない

### ログ確認

- [ ] `USER_SWITCH` 開始ログが出る
- [ ] Hive クリア完了ログが出る
- [ ] Windows の場合 `initializeForUser()` 完了ログが出る
- [ ] `UID保存完了` ログが B の UID で出る
- [ ] `ALL GROUPS` の currentUser が B の UID で出る

---

## テストケース 2: B → ログアウト → A

### 手順

- [ ] B でサインインした状態から開始する
- [ ] B 専用グループを確認する
- [ ] B 専用グループを選択する
- [ ] 任意の買い物リストを開く
- [ ] ログアウトする
- [ ] A でサインインする
- [ ] グループ一覧を開く

### 確認項目

- [ ] B 専用グループが表示されない
- [ ] A 専用グループが表示される
- [ ] B の最終選択グループが残っていない
- [ ] B の最終使用リストが残っていない

---

## テストケース 3: A → B → A 連続切替

### 手順

- [ ] A でサインインしてグループ一覧を確認する
- [ ] ログアウトして B でサインインする
- [ ] B のグループ一覧を確認する
- [ ] ログアウトして再び A でサインインする
- [ ] A のグループ一覧を確認する

### 確認項目

- [ ] 各切替後に表示グループが現在ユーザーの所属だけに一致する
- [ ] 切替を 2 回繰り返しても前ユーザーのグループ残留が再発しない
- [ ] グループ数が切替ごとに想定どおりになる

---

## テストケース 4: 同一ユーザー再ログイン

### 手順

- [ ] A でサインインする
- [ ] グループ一覧を確認する
- [ ] ログアウトする
- [ ] もう一度 A でサインインする

### 確認項目

- [ ] A のグループだけが表示される
- [ ] 他ユーザーのグループが混入しない
- [ ] 必要な同期後に表示が安定する

---

## テストケース 5: グループ 0 件ユーザーへの切替

### 手順

- [ ] グループ未所属のユーザー C を用意する
- [ ] A または B からログアウトして C でサインインする

### 確認項目

- [ ] 前ユーザーのグループが表示されない
- [ ] グループ一覧が 0 件になる
- [ ] 初回セットアップ導線またはグループタブ遷移が正しく動く

---

## Windows 専用確認項目

- [ ] ユーザー切替後に Windows の Hive フォルダ切替ログが出る
- [ ] `initializeForUser(newUid)` が切替ごとに実行される
- [ ] 別ユーザーに切り替えた直後から正しいグループだけが見える
- [ ] アプリ再起動後も直前ユーザーの状態だけが見える

---

## Android 実機確認項目

- [ ] A → B 切替で前ユーザーグループが残らない
- [ ] B → A 切替でも同じく残らない
- [ ] ログアウト後に UI が古いグループ一覧を一瞬保持し続けない

---

## NG 判定条件

- [ ] 前ユーザー専用グループが 1 件でも表示される
- [ ] 前ユーザーのグループが一瞬だけでも選択状態で開く
- [ ] 前ユーザーの買い物リストが初期表示される
- [ ] ログ上の currentUser UID と画面表示グループが一致しない
- [ ] 切替後に手動同期しないと正しいグループ一覧にならない
