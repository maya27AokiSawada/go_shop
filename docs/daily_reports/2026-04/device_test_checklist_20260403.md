## 2. 今回の重点確認項目（3/27〜28 新実装）

### 2.1 個人ボードを開く：多重タップ防止 + スピナー表示（`9e07e37`）

> **修正内容**: `member_tile_with_whiteboard.dart` を `ConsumerStatefulWidget` に変更し、
> `_isOpening` フラグで多重タップを防止。タイムアウトを 10秒→20秒 に延長。
> 読み込み中はリスト行末尾をスピナーに切り替える。

- [x] 個人ボードの行をタップすると末尾アイコンがスピナーに変わる
- [x] スピナー表示中に再タップしても重複してボードが開かない
- [x] ボードが開いたらスピナーが消える（アイコン表示に戻る）※ボードが開く速度が速くスピナーはほぼ見えない
- [N/A] ボードが開けなかった場合（タイムアウト等）もスピナーが消えてアイコンに戻る ※オフラインキャッシュ導入により、オフライン時もキャッシュからボードが開くため失敗シナリオを再現不可
- [x] 複数メンバーの行それぞれで独立したスピナー制御になっている（すもも→OK、まや→OK）

結果: OK

備考: スピナーはボードが開く速度が速いため実質見えないが、多重タップ防止と独立制御は正常動作確認。ボード開失敗のシナリオはオフラインキャッシュ導入により再現不可（キャッシュがある限りオフラインでも開く）。

### 2.2 終了時自動保存（全プラットフォーム・`silent` パラメータ対応）（`b06f370`）

> **修正内容**: 旧実装は Windows のみの自動保存。全プラットフォームで `_saveWhiteboard(silent: true)` を実行するよう拡張。
> `silent: true` のとき「保存しました」スナックバーを表示しない。

- [x] Android（SH-54D）で描画してから **保存ボタンを押さずに** 画面を閉じる
- [x] 閉じるとき「保存しました」スナックバーが **表示されない**（silent=true のため）
- [x] logcat に `💾 [AUTO_SAVE] 終了時自動保存を実行` が出る（10:42:07 確認）
- [x] 同じボードを再度開くと描画内容が保持されている
- [x] 他端末（Pixel 9）でも数秒以内に反映される
- [x] Pixel 9（まや）で同じシナリオを実施しても同様の結果（逆方向・SH-54Dへの反映も確認）

結果: OK

備考: SH-54D / Pixel 9 双方向で確認。logcat の `💾 [AUTO_SAVE] 終了時自動保存を実行`（10:42:07）確認済み。silent=true によりスナックバーなし・再入時の描画保持・他端末反映いずれも正常。

### 2.3 `PopScope` 移行（バックボタン自動保存）（`f256532`）

> **修正内容**: 廃止予定の `WillPopScope` を `PopScope` に移行。

- [x] Android バックボタンを押した際に `_navigateToGroupDetail` が呼ばれる（自動保存実行）

結果: OK

備考: バックボタン・バックジェスチャー両方で繰り返し確認。クラッシュ・無限ループなし。`PopScope` 移行後も動作に変化なし。
